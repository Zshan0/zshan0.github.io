Criteria for execution of a program:
For a system to execute a program, that program must be present on the main memory of the system, whenever you try to run a program, the loader module on your OS loads the program onto main memory and set PC(program counter) to the start of the program for it to run
Execution model: Processor only interacts with the main memory and does not interact with the hard drive or any other components, and when a program is to be runned. it fetches the instructions and data from the main memory.

The bottleneck/memory wall
For the processor to function fast and continuously it needs data from the main memory to be supplied to it at a reasonable speed. not only does the processor needs to be fast but also the main memory since processor needs the data for functioning
When the main memory is unable to keep up with the processor, the speed of processor slows down. this issue is called memory wall.

RAM
Large array of bytes Any address can be accessed in same time of memory
technologies we use for RAM Static RAM - low density (memory cells per area is small), high power consumption, expensive, fast access. contents will last till power supply (volatile) Dynamic RAM - high density, low power consumption, cheap, slower access needs refreshing(also volatile)
To design a single cell which stores in 1 bit in SRAM requires 4 - 6 transistors, hence they require more space, more power and are more expensive.
In DRAM, we need one capacitor and a transistor. charge - 1 no charge - 0 hence we need to refresh them periodically(10 to 100 milliseconds) so that the charge does not leave the capacitor. it also has sensitivity - can get effected by fields and corrupt the data.
where do we use SRAM and DRAM ?
SRAM - Cache memory (the memory between processor and the main memory which stores frequently used items) DRAM - Main memory and frame buffers.
Memory Hierarchy : Illusion of large and fast memory for the user
we use the program properties - temporal and spatial locality to achieve this locality
Principle of Locality The idea is that if something is accessed there is a high probability something nearby is also accessed or it will be accessed again.
Temporal locality(locality in time) - if an item is referenced then that same item will tend to be accessed again soon. Spatial Locality(locality in space) - if an item is referenced, then there's a chance that items near that (in address) will also tend be accessed.
Consider the following code:
for(int i = 0; i < n; i++) { sum += ar[i]; }
C++
Here i is being accessed again so this is temporal locality property, where as in array. the nearby elements are being accessed so it is spacial locality property.
Stride: The distance/gap between the current memory address accessed and the next memory accessed. in the above case it is the size of integer that is 4 bytes.
Linked lists vs Arrays
Suppose instead of the array in the above example we iterate over a linked list. the spacial locality for the linked lists will have a much larger stride than arrays since each node is spread across other regions of the RAM. hence the runtime for linked lists will have much greater time than compared to arrays.
General Cache Concept
The main memory is divided into small small blocks with some index on them, note that this partition is not physical but logical partitions. the size varies from arch to arch, but its usually of size 32/64 bytes Cache memory is also divided into same size of blocks.
Cache holds parts of the memory which are more frequently or more recently accessed for faster access in the future.
Cache hits and cache misses
Suppose the cache has some blocks of memory from the main memory - 1, 5, 9, 13.
now the processor wants some data which is present in the block 15 of the main memory.
So we need to access the data from not the cache but the main memory itself. The memory block 15 is brought to the cache and replaces some other block of memory if there is no space. This is considered as a CACHE MISS since cache was not of any use here. but suppose the processor wants to access some other data which is also present in block 15. now the data can directly be served to the processor from the cache since the block 15 is present in the cache, this is considered as a CACHE HIT.
Cache replacement policy When there is a cache miss, the block has to be fetched from the main memory and has to replace which are already in cache (block eviction). But which block to evict? this is answered by the CACHE REPLACEMENT POLICY.
Mapping the main memory blocks to the cache blocks
Consider the block size to be 32 bits. now that means that bits after the 6th bit will be same for any address in the same block. so those bits are the "tags" for the block. the first 4 bits are known as offset bits which display what memory it is referring to in each block.
Basic mapping strategy
Suppose there are 4 cache memory blocks.
So how do we map the main memory blocks to cache memory blocks? we take the tag of the block compute its modulo 4. that is we take the 1st 2 bits of the tag address and assign that main memory block to the cache memory block.
But then how do we check which block is present in that cache block? We give additional "meta-bits" to each cache block which represent which main memory block is stored there, based on that we can check if the cache block has the main memory block. In fact, we just have to store the meta-bits in each cache block to know if that block is present in the cache or not
Consider the following code:
for(i = 0; i < 128; i++) sum += A[i];
C++
Assuming the size of the array elements to be 4 bytes
The first element accessed A[0] is always considered a miss, also known as cold miss. now this block is moved to the cache, all elements up to A[7] are directly accessed from this block hence cache hits.
We notice a pattern here that multiples of 8 can be considered as cache hits. when A[32] is accessed, block 4 is moved to cache block 0 evicting the original block 0 present there.
More about mapping blocks to cache block:
index bits - which cache memory block each is mapped to tag bits - differentiate 2 main blocks mapped to the same cache block
Write hit policies: How writing is dealt when the block is present in the cache.
Write through policy: when there is a data update required on some memory, the changes are made on both respective cache block(if it is present) and the main memory block. There is no need for extra information to be stored here about update.
Write back policy: In this update system, we store another bit called the dirty bit. so when we make an update to block, it is only done to the respective cache block, after the update the dirty bit is switched. when a block with switched dirty bit needs to be evicted, it is written to main memory.
Write miss polices: When the block on which write is to be performed is absent from the cache.
Write allocate: Promote the required block to the cache and perform the write operation(just like read).
No-write allocate: Directly write to memory, no load to cache
Typical combinations of policies for writing are : Write through + no-write allocate Write back + write allocate (commonly used)
for(i = 0; i < 128; i += 32) sum += A[i]; for(i = 0; i < 128; i += 32) A[i] = -A[i];
C++
0, 32, 64 and 96 are all parts of memory blocks which are assigned to 0th cache block(%4 = 0)
All the reads can be considered are misses since every time we have to evict the block All the writes are hits since the read brings the required block to cache. Out of 12 access 8 are misses(all reads) and 4 are hits(all writes). this is a bad ratio. In this case only one cache block is being used.
Conflict misses: when 2 main blocks are mapped to the same cache block and end up evicting each other.
To fix so many conflict misses, we need to change how we map our main memory blocks to cache blocks.
Some organizations which reduce Conflict misses but with a trade off:
Capacity misses: misses which could have been avoided if we had large enough cache. that is, due to small size of cache we have to evict and replace again and again due to periodic access of blocks, this can be avoided if the cache size was big and the access again could have been directly from the cache.
Fully associative organization: Any main block can be allocated in any cache block. but the trade off is that to know what block is stored in the cache block, we need to store the address of the block rather than just tag bits(tag + index bits). this also takes more time to search for a block in the cache.
When a block is occupied, the main block to be moved is just moved to the next empty slot in the cache. if there are no empty slots, cache replacement policy is used to decide what block to remove.
Some common examples of Cache replacement policies are:
FIFO: first in first out LRU:least recently used LFU: least frequently used
Set associative organization: The set associative organization is in the grounds between fully associative organization and directly mapped organization. Cache is divided into n sets and m lines. A memory block is first mapped to a set, and then in that particular set it is mapped to a particular line.
How the block is placed?
The set is determined by the index bits of the memory block, and the cache line is determined by the tag bits of the block.
