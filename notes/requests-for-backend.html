Requests for backend
Important Note
Requests have to be made in such a way that they don't fetch all the data at once but piecewise based on the page number since all the data can not be carried to the frontend.
Route Format
request type URL {body content}
Image format
The image will be stored on the cloud and a link will be generated for each image. The URL will be stored and sent from the backend rather than the complete image.
So the image should be fetched from the link, this is to prevent the long waiting time for the website loading.
Authentication
With some specific requests, we are sending email and the password which must be stored locally after the login/signup.
After each specific request if the auth fails then they are redirected to the login page and the local storage are reset.
