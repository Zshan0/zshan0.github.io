Sequential Y86 Implementations
A computers speed is determined by its clock speed, that is the amount of time between 2 pulses
More than 1 instruction can be executed per clock cycle, it just depends on the processor
SEQ: Sequential processor
SEQ performs all the steps required to complete a instruction in 1 cycle.
Instruction stages in SEQ
1.
Fetch
2.
Decode
3.
Execute
4.
Memory
5.
Write-back
6.
PC update
Even though the instructions vary by a great scale, that is not all instructions need all the following steps.
Suppose there is an arithmetic operation which is performed on registers, that has nothing to do with memory.
But since making different pieces and requirements for each instruction just adds too much load onto hardware design. So to minimize the hardware complexity on the chip, we have a particular sequence for all the instructions.The idea is to use the same common circuits for all the instructions, It would be redundant to have 2 ALU on the same chip just because 2 operations which require ALU have different sequence of execution.
Stages:
Fetch:
In this piece of the instruction execution cycle, this stage reads the provided instruction.
the instruction is to be considered of 1 byte length, It has 2 pieces of 4 bit each.
icode: instruction code
ifun: instruction function
This could fetch a register specifier byte, one or both the registers, operand specifiers rA and rB.
Register specifier byte: Determines which register to pick, it is a byte long. so 8 digits. first 4 for choosing first operand register and the last 4 for the 2nd register
example: let the value of register specifier byte be 0F, so 0th register is picked in rA and F means no register is picked for rB.
There is also a possibility that it fetched a 4 byte constant.
In this step all the fetching is done so now it knows where the next instruction is going to start, that is it already knows what is the next position of PC. so it computes it and stores it, called valP.
valP = PC + (length of the fetched instruction)
Decode:
This part is for reading the register values for the 2 operands valA and valB.
In this phase most of the time it reads the instructions provided by the register specifier byte but sometimes it also reads stack pointer, example being pop.
Execute:
This stage is about performing operations,
performing arithmetic/logical operations based on the instruction code, calculate address of the memory required. increments or decrements the stack pointer. the resulting value is called valE.
The jump decision making also happens in this based on the condition statements.
Memory:
This stage might write data to memory, or read the data, the read data is valM
Write-back:
writes upto 2 results into the registers in 1 cycle
PC update:
based on valP calculated in fetch part the PC is updated.
