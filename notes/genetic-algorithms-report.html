Genetic Algorithms Report
Problem Statement
Given an overfit vector a dataset, use genetic algorithms to reduce the overfitting by eliminating the noise and getting a lower error.
The function is a linear function on 11 dimensions with mean squared error as the error function.
Constraints
The solution is a 11 dimensional vector of the form:
w=[w0,w2⋯w10]−10≤wi≤10  ∀i∈[0,10]\mathbf{w} = \begin{bmatrix} w_0, w_2 \cdots w_{10}\end{bmatrix}-10 \leq w_i \leq 10 \, \,\forall i \in [0, 10] w=[w0​,w2​⋯w10​​]−10≤wi​≤10∀i∈[0,10]
About Genetic Algorithms
Genetic Algorithms takes the approach similar to the natural evolution. It takes "Survival of the fittest" as an approach to find a solution according to our requirements.
The algorithm takes the same approach as natural selections, but in a very simple manner.
Population Description
A pool of vectors which satisfy the constraints described by the problem.
Individual Description
Each individual is a ordered list of values which is called a gene.
The Algorithm
Steps
In a broad sense, the algorithm is a loop with 3 abstract steps with a termination condition.
Initialization
A set of vectors are picked randomly or are given by the problem statement.
Mutation
A subset of the population undergoes changes in a subset of their genes.
Crossover
The population undergoes crossover pairwise where 2 individuals referred as parents create 2 other individuals referred as offsprings which take their genes from either of the parents based on the crossover technique.
Selection
The survival of an individual is based on its fitness, fitness is a function which maps an individual to a number. Higher fitness an individual has, higher chance of it surviving the selection. After selection the population left is the new generation.
Termination
The algorithm can be stopped after a generation if it satisfies the termination condition. The usual termination condition are either the number of generations or the convergence of the fitness.
Implementation Details
The base genetic algorithms which has been described above gives us the abstract understanding of the algorithm. The specifics of each step is dependent on the problem. We had tried multiple implementations for each step, first mentioning the ones which were finalized.

Mutation
For values of such scale with such large error values, mutation was the most crucial part of the algorithm, hence involved a lot of experimentation. Mentioned below is the final method that was used.
The best are unchanged
When the new generation is taken, it is already sorted based on the fitness function in the descending order. Rather than mutating the complete population, the best two individuals are remained untouched.
new_population = self.population[:2] # Top 2 remain unchanged. new_population.append(mutate_population(self.population[2:]) # -Remaining
Python
Due to small size of population, only the top 2 are picked
Gaussian Mutation
The value by which a gene must be mutated should be of the same order as that of the value itself. The the probability of mutation itself also must be not very large, larger the mutation, smaller should the probability be.
Gaussian distribution with appropriate standard deviation and mean should suffice these conditions.
•
The mutated value should be close to the original value since the mutation is small, which means μ=w\mu = wμ=w﻿ where www﻿ is the value of the gene.
•
The mutation value must of the same order as the value itself, σ=wscale\sigma = \frac{w}{scale}σ=scalew​﻿ where scale is a hyperparameter which has to be tuned.
•
The scale is the order by which the mutation is smaller than the value itself, for finer changes we set the scale to be high.
•
Since there is a strict bound over the weights, if the weights ever cross them after mutation, they are set to 0.
The above function was vectorized and was applied to the population in parallel.

Crossover
Crossover has a large impact when the population size is large, but in our case, we had to set it to be small. Each pair of crossover produces 2 offsprings.
Prioritizing the fittest
Rather than letting all the individuals be a part of the crossover, we only select the top half vectors for creating the next generation. But the process of picking the individuals out of this filtered population is random.
This is part of the selection since we are eliminating half of the population before making the offsprings

Selection
Selection happens based on the fitness values. In our implementation, the only form of selection that is happening is during the crossover when we eliminate half the population.
The main purpose of selection here is to sort the new offsprings based on their fitness values in descending order.
Saving the fit vector
Other than sorting, we also save the most fit function of the population, we compare it with the best vector so far and replace if it performs better.
This gives us the best vector throughout all the populations.
Fitness function
Fitness function is designed based on the problem statement and the requirements. For a simple linear regression problem with mean squared error problem, the fitness function can just be the negative of the train error.
Fitness function is more complicated when there are chances of falling in a local minima, but in the case of mean squared error there are no local minimas.
fitness(w)=−Train Error(w)\text{fitness}(\mathbf{w}) = -\text{Train Error}(\mathbf{w})fitness(w)=−Train Error(w)
Here Train Error is the 1st value returned by the server.
Termination
There is no definite termination condition defined for the genetic algorithm, rather it's the choice of the programmer. The termination conditions we used were the number of generations due to the limited number of queries.
def Termination(self): return self.generations >= MAX_GEN

Hyperparameters
Values were changed multiple times until the perfect value was found. All the hyperparameters are listed in parameters.py
There is one hyperparameter for each part of the algorithm.
•
Initialization: Population Size
•
Cross over: MAX_CROSS
•
Selection: MUTATE_TOP
•
Mutation: DIV_FACTOR
Population Size
The fixed size of the population through out the algorithm
Chosen value: 8
The value was initially set to 10, and then later tuned down to 8 since it didn't make a lot of difference and the queries were limited.
MAX_CROSS
Probability of the gene being interchanged individually for each position.
Chosen value: 0.9
Since the gene values were quite similar in the population due its small size, to have a larger impact due to crossover, values had to be interchanged more often.
MUTATE_TOP
The number of individuals that are picked from the top of the population for the next crossover.
Chosen value: 4
For population size of 8, 4 was the chosen value and for population size of 10 it was 5.
DIV_FACTOR
This is the factor by which the standard deviation is divided, as explained in the mutation.
Chosen value: 3
This value serves the same purpose as the step size, initially the value was large, and would be slowly tuned down as the function approached the minima.

Uncertain Survival
Probabilistic picking of the parents based on the fitness
•
Parents were sorted based on the fitness, descending order.
•
The probability of picking the later ones drops exponentially. Equivalent to binomial trails
def Crossover(self, population): probs = [BASE_PROB * (1 - BASE_PROB)**i for in range(self.population_size)] new_population = [] for _ in range(self.population_size / 2): parents = self.pick(population, probs) new_population.append(self.cross_parents(parents)) return new_population

United we Die
A strategy used to avoid local minima by making sure that the population is penalised for being close to each other which brings in diversity and allows the algorithm to explore more in the solution space.
Fitness(w)=−Train Error(w)−distance(w,picked)\text{Fitness}(\mathbf{w}) = - \text{Train Error}(\mathbf{w}) - \text{distance}(\mathbf{w}, \text{picked})Fitness(w)=−Train Error(w)−distance(w,picked)
Here distance calculates the sum of distance from all the individuals picked so far.

Removal of noise

Fitting on train error
Population is filled with underfit array and train error is used for fitness and is run for generations in batches of 10 with hyperparameters changed in between.
Hyperparameter Tuning
Changing the values after every 10 generations was done manually based on the pattern in train and test error for complete population.
Why does it work?
•
Since the noise which resulted in overfitting was eliminated, it made the weights underfit the data.
•
The weights were selected based on the training error, whereas the hyperparameter tuning was done based on the validation error.
•
This resulted in a good vector which performed well on unseen data.
